Concepto base

¿Qué es una Window Function?

Regular SQL vs Window Function
❌ GROUP BY (agrega, colapsa filas) Pierdes el detalle de cada fila. Solo ves el total del grupo.
✅ OVER() (calcula SIN colapsar) Cada fila conserva su información + el resultado del cálculo sobre el grupo.
SQL
-- La clave: OVER() define la "ventana" de filas sobre la que operar
SUM(importe) OVER (
    PARTITION BY cliente_id      -- agrupa (como GROUP BY pero sin colapsar)
    ORDER BY fecha               -- ordena dentro de cada grupo
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW  -- define el rango
)
Funciones de ranking

ROW_NUMBER · RANK · DENSE_RANK

SQL
SELECT cliente_id, ventas,
  ROW_NUMBER() OVER (ORDER BY ventas DESC) AS posicion,
  RANK()       OVER (ORDER BY ventas DESC) AS rank_con_hueco,
  DENSE_RANK() OVER (ORDER BY ventas DESC) AS rank_sin_hueco
FROM clientes

-- Resultado: ventas=100 → posicion=1, rank=1, dense_rank=1
--           ventas=100 → posicion=2, rank=1, dense_rank=1  (empate)
--           ventas=90  → posicion=3, rank=3, dense_rank=2  ← diferencia!
ROW_NUMBER()
Siempre único. En empates asigna números distintos (arbitrario).
Úsalo para: top-N, deduplicar
RANK()
Empates comparten posición. Deja huecos en el ranking.
Úsalo para: rankings deportivos
DENSE_RANK()
Empates comparten posición. Sin huecos en la secuencia.
Úsalo para: segmentos continuos
Funciones de desplazamiento

LAG y LEAD — comparar períodos

SQL
SELECT mes, ventas,
  LAG(ventas, 1) OVER (ORDER BY mes)   AS ventas_mes_anterior,
  LEAD(ventas, 1) OVER (ORDER BY mes)   AS ventas_mes_siguiente,
  ventas - LAG(ventas,1) OVER (ORDER BY mes) AS variacion_mom,
  ROUND(
    (ventas - LAG(ventas,1) OVER (ORDER BY mes))
    / NULLIF(LAG(ventas,1) OVER (ORDER BY mes), 0) * 100, 2
  ) AS variacion_pct
FROM metricas_mensuales

💡 LAG = miras atrás  ·  LEAD = miras adelante  ·  El segundo parámetro es cuántos pasos saltar

Funciones de agregación sobre ventana

Acumulados y medias
móviles con OVER()

SQL
SELECT fecha, importe,

  -- Acumulado total (YTD / running total)
  SUM(importe) OVER (ORDER BY fecha ROWS UNBOUNDED PRECEDING) AS acumulado,

  -- Media móvil 7 días
  AVG(importe) OVER (ORDER BY fecha ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) AS media_7d,

  -- % sobre el total del mes (sin GROUP BY)
  SUM(importe) OVER (PARTITION BY DATE_TRUNC(fecha,'month')) AS total_mes,
  ROUND(importe / SUM(importe) OVER (PARTITION BY DATE_TRUNC(fecha,'month')) * 100, 2) AS pct_mes

FROM transacciones
Segmentación

NTILE — cuartiles, deciles
y cualquier segmentación

SQL
SELECT cliente_id, total_compras,
  NTILE(4) OVER (ORDER BY total_compras DESC) AS cuartil,  -- Q1=top 25%
  NTILE(10) OVER (ORDER BY total_compras DESC) AS decil       -- D1=top 10%
FROM resumen_clientes

-- Uso real: segmentación RFM de clientes
SELECT cliente_id,
  CASE cuartil WHEN 1 THEN 'VIP'
              WHEN 2 THEN 'Alto valor'
              WHEN 3 THEN 'Medio'
              ELSE       'Bajo valor'
  END AS segmento
FROM (/* subquery con NTILE */)

💡 NTILE(n) divide el resultado en n grupos del mismo tamaño — perfecto para análisis RFM, scoring y KPIs de distribución

Valores de referencia

FIRST_VALUE y LAST_VALUE

SQL
SELECT producto_id, mes, precio,

  -- Precio del primer mes del año (base para calcular variación)
  FIRST_VALUE(precio) OVER (
    PARTITION BY producto_id ORDER BY mes
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
  ) AS precio_inicial,

  -- Precio más reciente de cada producto
  LAST_VALUE(precio) OVER (
    PARTITION BY producto_id ORDER BY mes
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
  ) AS precio_actual,

  -- Variación respecto al inicio
  ROUND((precio - FIRST_VALUE(precio) OVER(PARTITION BY producto_id ORDER BY mes
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING)) /
    NULLIF(FIRST_VALUE(precio) OVER(PARTITION BY producto_id ORDER BY mes
    ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING),0)*100,2) AS var_pct
FROM precios_historicos
En la práctica

8 casos de uso reales

1
ROW_NUMBER
Deduplicar registros: quedarte con la fila más reciente de cada cliente.
2
RANK
Top 5 productos por región sin subqueries anidados.
3
LAG
Calcular variación MoM (mes vs mes anterior) en métricas de negocio.
4
LEAD
Detectar churn: cliente que no compra en el siguiente período.
5
SUM OVER
Running total / YTD sin GROUP BY que pierda el detalle de fila.
6
AVG OVER
Media móvil 7/30 días para suavizar series temporales ruidosas.
7
NTILE
Segmentación RFM: clasificar clientes en cuartiles de valor.
8
FIRST_VALUE
Precio/valor base del período para calcular variaciones porcentuales.
